Operator's Guide

Everything the MashUp admin does, in one place — what each portal page is for, the order to run them in each week, and how to fix scorecards on the SimulatorGolfTour admin page when things go sideways.

Overview

The admin portal lives at /admin and is locked behind Cloudflare Access (Google sign-in). It has five working pages. Every change you save takes effect on the live site immediately — there's no separate publish step.

Two systems are in play and it helps to keep them straight:

  • MashUpThe admin portal on this site — seasons, events, team draws, handicaps, posters.
  • SGTSimulatorGolfTour's own admin page — registrations and scorecards. This is where you go to fix a broken round.

Weekly Workflow

The typical order of operations to run a weekly event start to finish.

  1. 1
    Create the tournament on SGT first. Our MashUp site can't do anything until the event exists on SGT. Go to SGT /admin → Tournaments → Create. Field-by-field settings are in Create Tournament below.
  2. 2
    Create the event on the Events page — paste the SGT event URL first, then pick the format.
  3. 3
    Refresh handicaps on the Players page so the draw uses current MashCAPs.
  4. 4
    Draw teams on the Teams page (Steps 1–4), then download the SGT Loading File.
  5. 5
    Upload the loading file to SGT to register the field for the tournament.
  6. 6
    Announce it on the Poster page — post to Discord, then activate the event (activation turns on live scorecard fetching).
  7. 7
    During play, fix any broken rounds on the SGT admin page as needed.

Events

Open Events →

Create and manage each week's event: format, tournament link, dates, and rounds.

  • •
    Paste the SGT Event URL first. It unlocks the rest of the form and auto-fills the event name, dates, round settings, and week number by scraping the SGT page.
  • •
    The event name is locked after the scrape — changing the format later won't rename it.
  • •
    Details opens a metadata panel (format, payouts, rounds) for active or completed events.
  • •
    ↺ Sync re-scrapes SGT to refresh dates and round settings on an existing event.
Activation matters: an event only fetches live scorecards once it's active. You normally activate it from the Poster page after announcing, but you can also do it here.

Players

Open Players →

The roster and the handicap engine. This is where MashCAPs get pulled and refreshed.

  • •
    Refresh Handicaps pulls every player's round history from SGT and recomputes their MashCAP — the average of their best 40% of rounds. This is the number that drives team draws and scoring.
  • •
    The table sorts by MashCAP (best to worst). A player without a MashCAP yet shows an amber ⚠ fallback using their raw SGT cap.
  • •
    Add, onboard, or remove players here. You can also set each player's Discord ID, which is used to @-tag winners in the results post.
SGT rate limit: the round-history API caps at roughly one full pull per 24 hours. If a refresh comes back missing players, you've likely hit the daily window — wait and retry later. Refresh before doing a team draw so the draw is balanced on current numbers.

Teams

Open Teams →

Draw teams for an event and produce the file SGT needs to register everyone. Four steps:

1
Create Teams

Three ways to build the draw:

  • –Tiered Draw — pulls one player from each handicap tier for balanced teams.
  • –Completely Random — everyone shuffled, pure luck.
  • –Manual Entry — click a player chip, then click a team slot to place them. Save unlocks once every slot is filled.
2
Generate SGT Loading File

Downloads the CSV used to register the field on SGT. It uses the season-scoped player list and the adjusted (relative) handicaps.

3
Upload SGT Loading File

Take the CSV over to the SGT admin page and upload it to register everyone for the tournament.

4
Configure Special Team Orders

For formats that need a fixed slot order (e.g. Lone Ranger). All teams are pre-loaded; use the ▲▼ buttons to reorder. Default is A = Slot 1, B = Slot 2, C = Slot 3.

Poster & Announce

Open Poster →

Generate the weekly announcement and post it to Discord.

  • •
    Renders a visual poster (exported as a PNG) plus the Discord announcement text — format rules, course settings, and prizes.
  • •
    One click posts the image and text to the Discord announcements channel.
  • •
    After posting it prompts you to activate the event — say yes to turn on live scorecard fetching for the week.

Seasons

Open Seasons →

Manage the season container that events and rosters live under.

  • •
    Create a new season and mark it active — the home page and handicap pages key off the active season.
  • •
    Assign the player roster for the season. The season roster is what scopes handicaps, draws, and the public handicap table.
  • •
    Archive completed seasons to keep the portal tidy while preserving their data.

SimulatorGolfTour Admin Page

These functions live on SGT's own admin page at www.simulatorgolftour.com/admin, not this portal. Create Tournament lives under Tournaments; the round-fixing tools live under the Game section. The golden rule for the Game tools:

Always screenshot the player's scorecard in SGT before you delete or modify anything, and double-check you've selected the correct Tournament and Round.
SGT · Tournaments

Create Tournament

This is the very first thing you do each week — the MashUp site can't create or sync an event until the tournament exists on SGT. Navigate to SGT /admin → Tournaments → Create. The form is extensive; settings below.

Naming standard: MASHUP - SXXWYY ABC — where SXX is the season, WYY the week, and ABC a short format description (e.g. "2 man scramble" or "3 man lone ranger").
Setting What to choose

Number of RoundsMost MashUp events are 1 round. Solo Ringer / special events are sometimes 2.
Registration OnSet to No — we don't want self-registration. We load players manually.
Rounds Count Toward Handicap?Yes, unless the format doesn't support handicaps (Scramble, Shamble, AltShot, etc.).
Rounds Count Toward Club Combo Handicap?N/A for us — doesn't matter.
Include in WGR?No.
Streams Required?No.
Points TypeAlways Tour — but it doesn't really matter since MashUp doesn't factor into WGR.
Game FormatChoose appropriately for our format — most often Stroke Play or Team Stroke Play.

SGT · Game

Reset Player

Resets a player inside an existing round back to a specific hole's tee box. For example, a player is on hole 5 and something totally wacky happens inside that hole — have them exit the round, then reset them to hole 5 so they can replay it cleanly.

SGT · Game

Delete Save Game

For when all hell breaks loose on a player inside a round. It can also completely delete a player's scorecard if they're forced to replay.

  • •
    Every time this has been used, you've had to choose YES to "Delete Scorecard". (It's unclear when you'd ever delete the game but keep the scorecard.)
  • •
    Be careful to pick the correct Tournament and Round to delete.
  • •
    You'll also need to run Update Player Resume afterward to put the player back in a playable state.
SGT · Game

Update Player Resume

Resets a player's playing status. Example: they completed Round 1 but forgot to stream, so they need to replay it.

  1. 1
    First use Delete Save Game to remove their scorecard.
  2. 2
    Then use this function to reset the player back to Round 1.
  3. 3
    Afterward, check View Registrations to confirm the player is "In Progress" or "Pending."
Watch the status: if a player shows "Completed," they will not see the tournament in their Tournaments window in GSPRO — so they can't replay until you reset them.
SGT · Game

Create / Modify Scorecard

Manually create a whole scorecard for a player, or update a single hole.

  • •
    Creating a full card is rare — maybe there's a good stream where you can see every score but no scorecard exists.
  • •
    Modifying is the common case — usually fixing a ghost read or a single bad hole.

To modify one hole:

  1. 1
    Confirm the right Tournament and Round.
  2. 2
    Change the first box to "UPDATE."
  3. 3
    Go to the hole in question and type in a new Gross and Net score.
Best practice: screenshot the card before changing it. Using UPDATE lets you modify just one hole without touching the rest.

Access & Deploys

  • •
    The admin portal is gated by Cloudflare Access (Google sign-in) — only approved accounts get in.
  • •
    Portal changes are live instantly (they write to storage, not code).
  • •
    Live scorecards fetch automatically every ~20 minutes, but only for active events.
